### 1. PineScript로 이동평균 전략 구현

**쉽게 이해하기**
- `ta.crossover(fast, slow)` 한 줄로 골든크로스를, `ta.crossunder(fast, slow)` 로 데드크로스를 감지할 수 있습니다.
- `label.new()` 로 신호 발생 시점에 차트에 화살표를 표시하면 시각적 검증이 쉬워집니다.

**미니 예시**
```pine
// PineScript v5 — 이동평균 골든크로스/데드크로스 전략
//@version=5
strategy("MA 골든크로스", overlay=true)

ma20 = ta.sma(close, 20)
ma60 = ta.sma(close, 60)

// 신호 조건
long_cond  = ta.crossover(ma20, ma60)   // 골든크로스 → 매수
short_cond = ta.crossunder(ma20, ma60)  // 데드크로스 → 청산

// 매매 실행
if long_cond
    strategy.entry("매수", strategy.long)
    label.new(bar_index, low,  "▲", style=label.style_label_up,   color=color.green)

if short_cond
    strategy.close("매수")
    label.new(bar_index, high, "▽", style=label.style_label_down, color=color.red)

// 이동평균 시각화
plot(ma20, color=color.blue,   linewidth=2, title="MA20")
plot(ma60, color=color.orange, linewidth=2, title="MA60")
```

### 2. PineScript 백테스트 결과 해석

**쉽게 이해하기**
- TradingView **전략 테스터** 탭의 Net Profit, Max Drawdown, Win Rate, Profit Factor, Sharpe Ratio 5가지를 기준값과 비교합니다.
- Profit Factor > 1.5, MDD > -15%, Sharpe > 1.0 을 동시에 만족해야 포트폴리오 제출 기준에 부합합니다.

**미니 예시**
```python
# TradingView 전략 테스터 결과 합격 기준 자동 체크
backtest = {
    'Net Profit (%)':   15.24,
    'Max Drawdown (%)': -8.3,
    'Win Rate (%)':     56.5,
    'Profit Factor':     1.8,
    'Sharpe Ratio':      1.12,
}
criteria = {
    'Net Profit (%)':   ('>0',    lambda v: v > 0),
    'Max Drawdown (%)': ('>-15%', lambda v: v > -15),
    'Win Rate (%)':     ('>50%',  lambda v: v > 50),
    'Profit Factor':    ('>1.5',  lambda v: v > 1.5),
    'Sharpe Ratio':     ('>1.0',  lambda v: v > 1.0),
}
print(f"{'지표':22s} {'값':>8s} {'기준':>8s}  결과")
print("-" * 52)
for key, val in backtest.items():
    crit_str, crit_fn = criteria[key]
    status = '✅ 합격' if crit_fn(val) else '⚠️ 미달'
    print(f"{key:22s} {val:>8.2f} {crit_str:>8s}  {status}")
```

### 3. Python 전략과 PineScript 결과 비교

**쉽게 이해하기**
- Python과 PineScript 결과가 완전히 일치하지 않을 수 있습니다. 데이터 소스·거래비용·체결 시점 가정이 다르기 때문입니다.
- 두 결과의 방향성(Sharpe 증감, MDD 변화)이 일치하면 전략이 일관성 있다고 볼 수 있습니다.

**미니 예시**
```python
# Python 백테스트 vs PineScript 전략 테스터 결과 비교
python_result = {
    'CAGR (%)':    8.5,
    'Sharpe':      1.08,
    'MDD (%)':    -9.2,
    'Win Rate (%)': 54.3,
}
pine_result = {
    'CAGR (%)':    8.3,    # 약간 낮음: 데이터·수수료 차이
    'Sharpe':      1.05,
    'MDD (%)':    -9.5,
    'Win Rate (%)': 53.8,
}
print(f"{'지표':15s} {'Python':>10s} {'PineScript':>12s} {'차이':>8s}")
print("-" * 48)
for key in python_result:
    py = python_result[key]
    ps = pine_result[key]
    print(f"{key:15s} {py:>10.2f} {ps:>12.2f} {py-ps:>+8.2f}")
```

### 4. 실습: 나만의 간단한 트레이딩 전략 PineScript 구현

**쉽게 이해하기**
- `ta.rsi()`, `ta.macd()`, `ta.bollinger_bands()` 등 내장 지표를 조합해 진입·청산 조건을 만들어보세요.
- 전략 테스터 결과에서 Profit Factor와 MDD를 우선 확인하고, 파라미터를 조정해 개선 방향을 찾습니다.

**미니 예시**
```pine
// PineScript v5 — RSI 역추세 전략 (과매도 매수 / 과매수 청산)
//@version=5
strategy("RSI 역추세", overlay=false, initial_capital=10000,
         commission_type=strategy.commission.percent, commission_value=0.1)

// 파라미터
rsi_len    = input.int(14, "RSI 기간",    minval=5,  maxval=30)
oversold   = input.int(30, "과매도 기준", minval=10, maxval=40)
overbought = input.int(70, "과매수 기준", minval=60, maxval=90)

rsi = ta.rsi(close, rsi_len)

// 시각화
plot(rsi, color=color.purple, title="RSI", linewidth=2)
hline(oversold,   "과매도", color=color.green, linestyle=hline.style_dashed)
hline(overbought, "과매수", color=color.red,   linestyle=hline.style_dashed)
hline(50,         "중립",   color=color.gray,  linestyle=hline.style_dotted)

// 매매 조건
if rsi < oversold
    strategy.entry("매수", strategy.long)   // 과매도 → 매수

if rsi > overbought
    strategy.close("매수")                  // 과매수 → 청산
```
