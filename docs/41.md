### 1. 백테스트(Backtest) 개념과 프레임워크

**쉽게 이해하기**
- **백테스트(Backtest) 개념과 프레임워크** 개념은 `일별 수익률 계산 → 누적 성과 확인 → 리스크 지표 비교`로 이해하면 쉽습니다.
- 먼저 작은 구간에서 계산이 맞는지 검증한 뒤, 전체 기간으로 확장하세요.

**미니 예시**
```python
# 백테스트(Backtest) 개념과 프레임워크 미니 예시
returns = [0.01, -0.02, 0.015, 0.005]
cum = 1.0
for r in returns:
    cum *= (1 + r)
print(f'누적수익률: {(cum - 1):.2%}')
```

### 2. MDD(Maximum Drawdown) 계산 및 해석

**쉽게 이해하기**
- **MDD(Maximum Drawdown)** 는 고점 대비 최대 낙폭으로, 전략이 경험한 최악의 손실 크기를 나타냅니다.
- -15% 이내를 합격선으로 보며, 값이 클수록 심리적·자금 리스크가 커집니다.

**미니 예시**
```python
# 고점 대비 최대 낙폭(MDD) 계산
prices = [100, 105, 103, 110, 95, 98, 102]
peak = prices[0]
mdd = 0.0
for p in prices:
    if p > peak:
        peak = p
    drawdown = (p - peak) / peak
    mdd = min(mdd, drawdown)
print(f'MDD: {mdd:.2%}')  # MDD: -13.64%
```

### 3. 샤프 비율(Sharpe Ratio) 계산

**쉽게 이해하기**
- **샤프 비율(Sharpe Ratio)** = (연평균수익 − 무위험수익) / 연환산변동성으로, 1.0 이상이면 합격선입니다.
- 수익률이 같아도 변동성이 크면 샤프 비율이 낮으므로 안정성 비교에 적합합니다.

**미니 예시**
```python
# 연환산 샤프 비율 계산 (무위험수익률 연 3% 가정)
import math
returns = [0.01, -0.02, 0.015, 0.005, -0.01, 0.02]
rf_daily = 0.03 / 252
excess = [r - rf_daily for r in returns]
mean_e = sum(excess) / len(excess)
var_e = sum((e - mean_e) ** 2 for e in excess) / len(excess)
sharpe = (mean_e / math.sqrt(var_e)) * math.sqrt(252)
print(f'샤프 비율: {sharpe:.2f}')  # 1.0 이상이면 합격
```

### 4. 칼마 비율(Calmar Ratio), 승률(Win Rate)

**쉽게 이해하기**
- **승률** = 수익 거래 수 / 전체 거래 수, **칼마 비율** = 연평균수익률 / |MDD| 로 전략 안정성을 확인합니다.
- 승률이 낮아도 손익비가 높으면 수익이 날 수 있으므로 두 지표를 함께 봐야 합니다.

**미니 예시**
```python
# 승률과 칼마 비율 계산
returns = [0.01, -0.02, 0.015, 0.005, -0.01, 0.02]
win_rate = sum(1 for r in returns if r > 0) / len(returns)
print(f'승률: {win_rate:.2%}')  # 승률: 66.67%

# 칼마 비율 = 연평균수익률 / |MDD|
annual_ret = sum(returns) / len(returns) * 252
prices = [1.0]
for r in returns:
    prices.append(prices[-1] * (1 + r))
peak = prices[0]
mdd = 0.0
for p in prices:
    if p > peak:
        peak = p
    mdd = min(mdd, (p - peak) / peak)
calmar = annual_ret / abs(mdd)
print(f'칼마 비율: {calmar:.2f}')
```

### 5. 실습: vectorbt / backtrader로 백테스트 실행

**쉽게 이해하기**
- **vectorbt**는 pandas 확장으로 빠른 벡터 연산 백테스트, **backtrader**는 이벤트 기반으로 복잡한 전략 구현에 적합합니다.
- 라이브러리 없이도 `position = signal.shift(1)` 패턴으로 직접 구현할 수 있습니다.

**미니 예시**
```python
# vectorbt 없이 직접 구현하는 MA 크로스오버 백테스트
import numpy as np

# 가상 일별 수익률과 포지션 신호
returns = [0.01, -0.02, 0.015, -0.005, 0.02, 0.008, -0.01, 0.012]
signal  = [1,    1,     0,      0,      1,    1,      1,     1   ]  # 1=보유, 0=현금

# 전략 수익률 = 수익률 × 전날 포지션 (미래 데이터 방지)
strat = [r * s for r, s in zip(returns[1:], signal[:-1])]

cum = 1.0
for r in strat:
    cum *= (1 + r)
sharpe = np.mean(strat) / np.std(strat) * np.sqrt(252)
print(f'전략 누적수익률: {(cum - 1):.2%}')
print(f'샤프 비율:       {sharpe:.2f}')
```

# Day 058 — 성과 개선 및 리밸런싱

> **모듈 9: 데이터 활용 퀀트 모델링** | 2/5일차 | ⚙️ | 학습시간: 8시간

- 리밸런싱 주기별 성과 비교
- 포지션 사이징(Kelly Criterion 등)

### 1. 과최적화(Overfitting) 방지: 워크포워드 분석

**쉽게 이해하기**
- **워크포워드(Walk-Forward)** 는 훈련 기간과 테스트 기간을 시간 순으로 분리해 순차적으로 검증하는 방법입니다.
- In-sample에서만 잘 맞는 파라미터가 Out-of-sample에서도 유효한지 반드시 확인해야 합니다.

**미니 예시**
```python
# 워크포워드: 훈련/테스트 기간을 순차적으로 분리
years = list(range(2015, 2025))
train_window = 3  # 3년 학습 윈도우

for i in range(train_window, len(years)):
    train = years[i - train_window: i]
    test  = years[i]
    print(f'학습: {train[0]}~{train[-1]} → 테스트: {test}')
# 출력 예시:
# 학습: 2015~2017 → 테스트: 2018
# 학습: 2016~2018 → 테스트: 2019
# 학습: 2017~2019 → 테스트: 2020
```

### 2. 거래비용(슬리피지, 수수료) 반영

**쉽게 이해하기**
- **수수료(Commission)** 는 거래당 고정 비용, **슬리피지(Slippage)** 는 호가 스프레드로 인한 불리한 체결 비용입니다.
- 거래비용을 반영하지 않으면 백테스트 수익률이 실제보다 과대 계산됩니다.

**미니 예시**
```python
# 거래비용 반영 — 수수료 + 슬리피지
commission = 0.001   # 0.1% 수수료
slippage   = 0.0005  # 0.05% 슬리피지
cost_per_trade = commission + slippage

returns = [0.01, -0.02, 0.015, 0.005]
trade   = [1, 0, 1, 1]   # 매매 발생 여부 (1=매매, 0=보유)
adj     = [r - cost_per_trade * t for r, t in zip(returns, trade)]

print('비용 전:', [f'{r:.3%}' for r in returns])
print('비용 후:', [f'{r:.3%}' for r in adj])
# 비용 후 수익률이 낮아지는 것을 확인
```

### 3. 리밸런싱 주기별 성과 비교

**쉽게 이해하기**
- 리밸런싱을 자주 할수록 목표 비중 유지에 유리하지만 거래비용이 누적됩니다.
- **월별 vs 분기별 vs 연간** 리밸런싱의 비용 차이를 먼저 계산하고, 실제 성과와 비교하세요.

**미니 예시**
```python
# 리밸런싱 주기별 연간 거래비용 비교
years = 3
options = {'월별': 12, '분기별': 4, '반기별': 2, '연간': 1}
cost_per_trade = 0.001  # 수수료 0.1%

for label, freq in options.items():
    annual_cost = freq * cost_per_trade
    total_cost  = annual_cost * years
    print(f'{label:5s} ({freq:2d}회/년): 연간비용 {annual_cost:.2%}, {years}년 누적 {total_cost:.2%}')
```

### 4. 포지션 사이징(Kelly Criterion 등)

**쉽게 이해하기**
- **켈리 공식** f\* = (b·p − q) / b 로 최적 베팅 비율을 계산합니다 (b=손익비, p=승률, q=패율).
- 실전에서는 변동성을 줄이기 위해 켈리 비율의 절반(반켈리)을 적용하는 것이 일반적입니다.

**미니 예시**
```python
# 켈리 공식으로 최적 포지션 비율 계산
win_rate = 0.55    # 승률 55%
avg_win  = 0.02    # 평균 수익 2%
avg_loss = 0.01    # 평균 손실 1%
b = avg_win / avg_loss  # 손익비 = 2.0

kelly = (b * win_rate - (1 - win_rate)) / b
half_kelly = kelly * 0.5  # 반켈리: 안전 마진 확보

print(f'손익비(b):       {b:.1f}')
print(f'풀켈리 비율:     {kelly:.2%}')
print(f'반켈리(권장):    {half_kelly:.2%}')
```

### 5. 실습: 리밸런싱 전략 개선 실습

**쉽게 이해하기**
- 거래비용, 워크포워드, 켈리 비율을 모두 적용한 뒤 성과 지표가 개선되는지 확인합니다.
- 개선 전후의 Sharpe, MDD, 승률을 비교 테이블로 정리하면 포트폴리오에 설득력이 생깁니다.

**미니 예시**
```python
# 개선 전후 성과 지표 비교 템플릿
before = {'CAGR': 0.082, 'Sharpe': 0.95, 'MDD': -0.18}
after  = {'CAGR': 0.079, 'Sharpe': 1.12, 'MDD': -0.12}

print(f"{'지표':10s} {'개선 전':>10s} {'개선 후':>10s} {'변화':>10s}")
print("-" * 44)
for key in before:
    b, a = before[key], after[key]
    diff = a - b
    sign = '+' if diff >= 0 else ''
    print(f"{key:10s} {b:>10.2%} {a:>10.2%} {sign}{diff:>9.2%}")
```
